Just a small correction : - On older Seagate drives (ST-10 type and older) the adaptives are stored on the platters and because of that it's possible to replace the PCB of the drive with a compatible one without the need of replacing or re-programming the ROM chip as long as the ROM code is the same. ROM must match app code on the platter but if the firmware version of the donor is the same as the firmware version of the patient there should be no issue replacing the PCB as both ROM codes will be the same. Only on F3 arch drives - 7200.11 and newer Seagate did start storing the adaptives on the ROM chip (or embedded ROM) on the PCB.
